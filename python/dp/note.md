# DP 란?
정의 : 문제에 대한 정답이 될 가능성이 있는 모든 해결책을 "체계적"이고 "효율적"으로 탑색하는 풀이법

1. 크고 복잡한 문제를 작은 문제로 나눈다.(subproblem)
2. 하위 문제의 답 계산
    * 중복 계산해야하는 하위문제(overlapping subproblem)이 있다면 재계산 하지 않도록 한다 -> memoization, dp table
3. 하위 문제의 답을 통해 원래 문제인 큰 문제의 답을 도출한다.
    * 최적 부분 구조(optimal substurcture) : 하위 부분문제에서 구한 답으로 큰 문제의 최적의 답을 구할수 있는 구조
4. 재귀를 사용하기 떄문에 recurrence relation(점화식), base case 가 필요하다

# top down & bottom up 의 특징(fibo.py 참고)
탑다운 방식: 
1. 재귀를 사용하여 구현이 빠르다.
2. 함수를 계속 호출해야 하기에 메모리 오버해드가 발생하여 실행시간이 조금 느릴수 있다.
3. 재귀풀이에서 중복된 계산 값을 저장(memoization)하여 재사용한다.
바텀업 방식: 
1. 반복문을 사용해서 실행시간이 빠르다
2. subproblem에 대한 계산 값을 DB table 에 저장(tabulation)하여 계산에 활용한다.
